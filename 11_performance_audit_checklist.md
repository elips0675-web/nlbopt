# Performance Audit Checklist for 1C-Bitrix

## 1. Изображения (~7 МБ — главная проблема!)

### Что проверять

| Файл/механизм | Что проверить |
|---------------|---------------|
| Настройки инфоблоков Битрикс | Размеры `resize_cache`, генерация WebP/AVIF |
| `/bitrix/php_interface/` или `/local/php_interface/` | Кастомные обработчики ресайза изображений |
| Шаблоны компонентов | Вывод картинок — `CFile::ResizeImageGet()` или прямой URL |
| `init.php` | Хуки на обработку изображений |

### Конкретные проблемы

- **JPEG вместо WebP/AVIF** — почти все картинки в JPEG
- **Избыточное разрешение** — картинки 752×450 отображаются как 694×391
- **Нет srcset** — нет адаптивных изображений под разные экраны
- **Фоновые картинки через `background-image`** — не лениво загружаются, блокируют рендер

## 2. JavaScript (второй по весу)

| Файл | Что искать |
|------|------------|
| `/bitrix/js/` или `/local/js/` | Сторонние библиотеки (jQuery, OwlCarousel и т.д.) |
| Шаблоны с OwlCarousel | `ul.owl-stage` — старая версия |
| `template_styles.css` / `styles.css` | Inline-скрипты, блокирующие рендер |
| `header.php` шаблона | Скрипты в `<head>` вместо перед `</body>` |

---

## 3. CSS

| Файл | Что искать |
|------|------------|
| `template_styles.css` | Неиспользуемые стили (Bootstrap/jQuery UI) |
| `/bitrix/css/` | Стили ядра Битрикс, которые можно отключить |
| Inline-стили в шаблонах | `style=""` в компонентах — заменить на классы |

---

## 4. Сервер / Кэширование

| Файл | Что проверить |
|------|---------------|
| `.htaccess` | Сжатие (gzip/brotli), кэширование статики, заголовки для изображений |
| `/bitrix/.settings.php` | Настройки кэширования, композитного режима |
| `/bitrix/php_interface/after_connect.php` или `dbconn.php` | Настройки БД, медленные запросы |
| Настройки хостинга / Nginx | HTTP/2 или HTTP/3, Brotli, кэширование |

---

## 5. Шрифты и сторонние ресурсы

| Файл | Что искать |
|------|------------|
| `header.php` шаблона | Google Fonts или CDN — preconnect, `display=swap` |
| Внешние скрипты | Метрики, аналитика, виджеты — отложенная загрузка |

---

## Итоговый чеклист файлов для анализа

```
1. .htaccess                          ← сжатие, кэш, заголовки
2. /bitrix/.settings.php              ← композит, кэш, производительность
3. /local/templates/{тема}/header.php ← порядок загрузки ресурсов
4. /local/templates/{тема}/footer.php ← скрипты в конце
5. /local/templates/{тема}/template_styles.css
6. /local/php_interface/init.php      ← хуки, обработчики
7. Шаблоны компонентов с картинками   ← CFile::ResizeImageGet, srcset
8. Настройки инфоблоков (админка)     ← размеры resize_cache
```

**Главное: изображения дают ~80% проблемы.** Самый быстрый выигрыш — настроить автоматическую конвертацию в WebP и правильные размеры `resize_cache`. В новых версиях Битрикс есть встроенная опция «Конвертация изображений в WebP» в настройках главного модуля.
